What is Redux?

Definition: Redux is a predictable state container for JavaScript apps. It helps you manage the state (data) of your application in a single place.

Simple Explanation: Imagine a big warehouse (Store) where all your app's data is kept. Instead of each room (component) having its own storage, everyone goes to the warehouse to get or update things.

üè™ The Warehouse Analogy

  • Store: The warehouse building
  • State: All items in the warehouse
  • Actions: Request forms (like "Give me item X")
  • Reducers: Warehouse workers who process requests
  • Dispatch: Submitting the request form

Core Principles

1Ô∏è‚É£ Single Source of Truth +

Definition: The entire state of your application is stored in one object tree within a single store.

Simple: All data lives in one place, not scattered everywhere.

// The entire app state in one object
const state = {
    user: { name: "Rajneesh", loggedIn: true },
    todos: ["Learn Redux", "Build Project"],
    cart: { items: [], total: 0 }
};
2Ô∏è‚É£ State is Read-Only +

Definition: The only way to change state is to emit an action, an object describing what happened.

Simple: You can't directly change data. You must send a request (action) to change it.

// ‚ùå Wrong - Direct mutation
state.user.name = "New Name";

// ‚úÖ Correct - Dispatch an action
dispatch({ 
    type: "UPDATE_NAME", 
    payload: "New Name" 
});
3Ô∏è‚É£ Changes are Made with Pure Functions +

Definition: Reducers are pure functions that take the previous state and an action, and return the next state.

Pure Function: Same input always gives same output, no side effects. 

// Pure function - predictable
function add(a, b) {
    return a + b;  // Always same result for same inputs
}

// Impure function - unpredictable
function addRandom(a) {
    return a + Math.random();  // Different result each time
}

// Redux reducer (pure function)
function counterReducer(state = 0, action) {
    if (action.type === "INCREMENT") {
        return state + 1;
    }
    return state;
}

Redux Data Flow (The Journey)

üîÑ Step-by-Step Flow

  1. User clicks button ‚Üí Triggers an event
  2. Dispatch an Action ‚Üí Send request to store
  3. Reducer processes ‚Üí Updates state
  4. Store updates ‚Üí New state created
  5. UI re-renders ‚Üí Shows new data
// 1. Define Action Types
const INCREMENT = "INCREMENT";
const DECREMENT = "DECREMENT";

// 2. Create Actions (Request forms)
const increment = () => ({ type: INCREMENT });
const decrement = () => ({ type: DECREMENT });

// 3. Create Reducer (Warehouse worker)
function counterReducer(state = 0, action) {
    switch(action.type) {
        case INCREMENT:
            return state + 1;
        case DECREMENT:
            return state - 1;
        default:
            return state;
    }
}

// 4. Create Store (Warehouse)
import { createStore } from 'redux';
const store = createStore(counterReducer);

// 5. Subscribe to changes (Get notifications)
store.subscribe(() => {
    console.log("State changed:", store.getState());
});

// 6. Dispatch actions (Submit requests)
store.dispatch(increment()); // State: 1
store.dispatch(increment()); // State: 2
store.dispatch(decrement()); // State: 1

// 1. ACTION TYPES (Constants)
const ADD_TODO = "ADD_TODO";
const TOGGLE_TODO = "TOGGLE_TODO";
const DELETE_TODO = "DELETE_TODO";

// 2. ACTION CREATORS (Functions that return actions)
const addTodo = (text) => ({
    type: ADD_TODO,
    payload: {
        id: Date.now(),
        text: text,
        completed: false
    }
});

const toggleTodo = (id) => ({
    type: TOGGLE_TODO,
    payload: id
});

const deleteTodo = (id) => ({
    type: DELETE_TODO,
    payload: id
});

// 3. REDUCER (Pure function that updates state)
const initialState = {
    todos: []
};

function todoReducer(state = initialState, action) {
    switch(action.type) {
        case ADD_TODO:
            return {
                ...state,
                todos: [...state.todos, action.payload]
            };
        
        case TOGGLE_TODO:
            return {
                ...state,
                todos: state.todos.map(todo =>
                    todo.id === action.payload
                        ? { ...todo, completed: !todo.completed }
                        : todo
                )
            };
        
        case DELETE_TODO:
            return {
                ...state,
                todos: state.todos.filter(todo => todo.id !== action.payload)
            };
        
        default:
            return state;
    }
}

// 4. CREATE STORE
import { createStore } from 'redux';
const store = createStore(todoReducer);

// 5. SUBSCRIBE (Listen to changes)
store.subscribe(() => {
    console.log("Current State:", store.getState());
});

// 6. DISPATCH ACTIONS (Use the store)
store.dispatch(addTodo("Learn Redux"));
store.dispatch(addTodo("Build Project"));
store.dispatch(toggleTodo(1));
store.dispatch(deleteTodo(2));

console.log("Final State:", store.getState());

Why Redux Toolkit?

Definition: Redux Toolkit (RTK) is the official, recommended way to write Redux logic. It simplifies Redux code and includes best practices by default.

üéØ Problems RTK Solves

  • Too much boilerplate: Old Redux needed lots of repetitive code
  • Complex setup: Setting up store was complicated
  • Immutability: Hard to update state without mutations
  • Async logic: Handling API calls was tricky

‚ú® RTK Benefits

  • Less code: Write 50% less code
  • Built-in Immer: Write "mutating" code that's actually immutable
  • Built-in Thunks: Easy async operations
  • DevTools: Automatic Redux DevTools setup

2Ô∏è‚É£ createSlice() +

Definition: A function that accepts an initial state, an object of reducer functions, and a "slice name", and automatically generates action creators and action types.

Simple: One function to create actions and reducers together!

import { createSlice } from '@reduxjs/toolkit';

// Create a slice (piece of state)
const counterSlice = createSlice({
    name: 'counter',  // Name of this slice
    initialState: {
        value: 0
    },
    reducers: {
        // Actions are automatically created!
        increment: (state) => {
            // Looks like mutation, but Immer makes it immutable
            state.value += 1;
        },
        decrement: (state) => {
            state.value -= 1;
        },
        incrementByAmount: (state, action) => {
            state.value += action.payload;
        },
        reset: (state) => {
            state.value = 0;
        }
    }
});

// Export actions (automatically created)
export const { increment, decrement, incrementByAmount, reset } = counterSlice.actions;

// Export reducer
export default counterSlice.reducer;

// import { increment } from './counterSlice';

3Ô∏è‚É£ createAsyncThunk() +

Definition: A function that accepts a Redux action type string and a callback function that returns a promise. It generates promise lifecycle action types based on the action type prefix.

Simple: Easy way to handle API calls in Redux!

import { createSlice, createAsyncThunk } from '@reduxjs/toolkit';
import axios from 'axios';

// Create async thunk for API call
export const fetchUsers = createAsyncThunk(
    'users/fetchUsers',  // Action type prefix
    async () => {
        const response = await axios.get('https://api.example.com/users');
        return response.data;  // This becomes action.payload
    }
);

const usersSlice = createSlice({
    name: 'users',
    initialState: {
        users: [],
        loading: false,
        error: null
    },
    reducers: {},
    extraReducers: (builder) => {
        builder
            // When API call starts
            .addCase(fetchUsers.pending, (state) => {
                state.loading = true;
                state.error = null;
            })
            // When API call succeeds
            .addCase(fetchUsers.fulfilled, (state, action) => {
                state.loading = false;
                state.users = action.payload;
            })
            // When API call fails
            .addCase(fetchUsers.rejected, (state, action) => {
                state.loading = false;
                state.error = action.error.message;
            });
    }
});

export default usersSlice.reducer;

// Usage in component:
// dispatch(fetchUsers());

Understanding Async Operations

Definition: Asynchronous operations are tasks that take time to complete, like fetching data from an API. Redux Toolkit provides createAsyncThunk to handle these operations.

üéØ Three States of Async Operations

  • Pending: Request is in progress (show loading spinner)
  • Fulfilled: Request succeeded (show data)
  • Rejected: Request failed (show error message)

Complete Async Example - User Management

// ========== usersSlice.js ==========
import { createSlice, createAsyncThunk } from '@reduxjs/toolkit';
import axios from 'axios';

const API_URL = 'https://jsonplaceholder.typicode.com/users';

// Async thunk to fetch users
export const fetchUsers = createAsyncThunk(
    'users/fetchUsers',
    async (_, { rejectWithValue }) => {
        try {
            const response = await axios.get(API_URL);
            return response.data;
        } catch (error) {
            return rejectWithValue(error.response.data);
        }
    }
);

// Async thunk to add user
export const addUser = createAsyncThunk(
    'users/addUser',
    async (userData, { rejectWithValue }) => {
        try {
            const response = await axios.post(API_URL, userData);
            return response.data;
        } catch (error) {
            return rejectWithValue(error.response.data);
        }
    }
);

// Async thunk to update user
export const updateUser = createAsyncThunk(
    'users/updateUser',
    async ({ id, userData }, { rejectWithValue }) => {
        try {
            const response = await axios.put(`${API_URL}/${id}`, userData);
            return response.data;
        } catch (error) {
            return rejectWithValue(error.response.data);
        }
    }
);

// Async thunk to delete user
export const deleteUser = createAsyncThunk(
    'users/deleteUser',
    async (id, { rejectWithValue }) => {
        try {
            await axios.delete(`${API_URL}/${id}`);
            return id;
        } catch (error) {
            return rejectWithValue(error.response.data);
        }
    }
);

const usersSlice = createSlice({
    name: 'users',
    initialState: {
        users: [],
        loading: false,
        error: null,
        success: false
    },
    reducers: {
        clearError: (state) => {
            state.error = null;
        },
        clearSuccess: (state) => {
            state.success = false;
        }
    },
    extraReducers: (builder) => {
        builder
            // Fetch Users
            .addCase(fetchUsers.pending, (state) => {
                state.loading = true;
                state.error = null;
            })
            .addCase(fetchUsers.fulfilled, (state, action) => {
                state.loading = false;
                state.users = action.payload;
            })
            .addCase(fetchUsers.rejected, (state, action) => {
                state.loading = false;
                state.error = action.payload || 'Failed to fetch users';
            })
            
            // Add User
            .addCase(addUser.pending, (state) => {
                state.loading = true;
            })
            .addCase(addUser.fulfilled, (state, action) => {
                state.loading = false;
                state.users.push(action.payload);
                state.success = true;
            })
            .addCase(addUser.rejected, (state, action) => {
                state.loading = false;
                state.error = action.payload || 'Failed to add user';
            })
            
            // Update User
            .addCase(updateUser.fulfilled, (state, action) => {
                const index = state.users.findIndex(u => u.id === action.payload.id);
                if (index !== -1) {
                    state.users[index] = action.payload;
                }
                state.success = true;
            })

            .addCase(deleteUser.fulfilled, (state, action) => {
                state.users = state.users.filter(u => u.id !== action.payload);
                state.success = true;
            });
    }
});

Error Handling Best Practices

// ========== Advanced Error Handling ==========

// 1. Create a custom error handler
export const handleAsyncError = (error) => {
    if (error.response) {
        // Server responded with error
        return {
            message: error.response.data.message || 'Server error',
            status: error.response.status
        };
    } else if (error.request) {
        // Request made but no response
        return {
            message: 'No response from server',
            status: 0
        };
    } else {
        // Something else happened
        return {
            message: error.message || 'Unknown error',
            status: -1
        };
    }
};

// 2. Use in async thunk
export const fetchData = createAsyncThunk(

        try {
            const response = await axios.get('/api/data');
            return response.data;
        } catch (error) {
            const errorData = handleAsyncError(error);
            return rejectWithValue(errorData);
        }
    }
);

1. Normalized State

Definition: Normalized state means storing data in a flat structure with IDs as keys, instead of nested arrays and objects. This makes updates faster and easier.

‚ùå Non-Normalized (Bad)

const state = {
    posts: [
        {
            id: 1,
            title: "Post 1",
            author: { id: 1, name: "Rajneesh" },
            comments: [
                { id: 1, text: "Great!" },
                { id: 2, text: "Nice!" }
            ]
        },
        {
            id: 2,
            title: "Post 2",
            author: { id: 1, name: "Rajneesh" },  // Duplicate!
            comments: []
        }
    ]
};

// Problem: To update author name, you need to update multiple places!

‚úÖ Normalized (Good)

const state = {
    posts: {
        ids: [1, 2],
        entities: {
            1: { id: 1, title: "Post 1", authorId: 1, commentIds: [1, 2] },
            2: { id: 2, title: "Post 2", authorId: 1, commentIds: [] }
        }
    },
    authors: {
        ids: [1],
        entities: {
            1: { id: 1, name: "Rajneesh" }
        }
    },
    comments: {
        ids: [1, 2],
        entities: {
            1: { id: 1, text: "Great!" },
            2: { id: 2, text: "Nice!" }
        }
    }
};

// Benefit: Update author once, reflects everywhere!

2. createEntityAdapter

Definition: A function that generates a set of prebuilt reducers and selectors for performing CRUD operations on normalized state. 
// ========== postsSlice.js with Entity Adapter ==========
import { createSlice, createEntityAdapter, createAsyncThunk } from '@reduxjs/toolkit';
import axios from 'axios';

// Create entity adapter
const postsAdapter = createEntityAdapter({
    // Assume IDs are stored in 'id' field (default)
    selectId: (post) => post.id,
    // Sort by title
    sortComparer: (a, b) => a.title.localeCompare(b.title)
});

// Get initial state from adapter
const initialState = postsAdapter.getInitialState({
    loading: false,

4. Custom Middleware

Definition: Middleware is code that runs between dispatching an action and the action reaching the reducer. It's used for logging, crash reporting, async operations, etc. 
// ========== Custom Middleware Examples ==========

// 1. Logger Middleware
const loggerMiddleware = (store) => (next) => (action) => {
    console.group(action.type);
    console.log('Previous State:', store.getState());
    console.log('Action:', action);
    
    const result = next(action);
    
    console.log('Next State:', store.getState());
    console.groupEnd();
    
    return result;
};

// 2. Error Handler Middleware
const errorMiddleware = (store) => (next) => (action) => {
    try {
        return next(action);
    } catch (error) {
        console.error('Error in action:', action.type, error);
        // Send to error tracking service
        // Sentry.captureException(error);
        throw error;
    }
};

// 3. API Middleware
const apiMiddleware = (store) => (next) => (action) => {
    if (action.type !== 'API_REQUEST') {
        return next(action);
    }
    
    const { url, method, data, onSuccess, onError } = action.payload;
    
    fetch(url, { method, body: JSON.stringify(data) })
        .then(res => res.json())
        .then(data => store.dispatch({ type: onSuccess, payload: data }))
        .catch(error => store.dispatch({ type: onError, payload: error }));
};

// 4. Local Storage Middleware (Persistence)
const localStorageMiddleware = (store) => (next) => (action) => {
    const result = next(action);
    
    // Save specific slices to localStorage
    const state = store.getState();
    localStorage.setItem('reduxState', JSON.stringify({
        user: state.user,
        cart: state.cart
    }));
    
    return result;
};

// Add middleware to store

const store = configureStore({
    reducer: rootReducer,
    middleware: (getDefaultMiddleware) =>
        getDefaultMiddleware().concat(
            loggerMiddleware,
            errorMiddleware,
            apiMiddleware,
            localStorageMiddleware
        )
});

export const useAsyncAction = () => {
    const dispatch = useAppDispatch();
    const [loading, setLoading] = useState(false);
    const [error, setError] = useState(null);

    const execute = async (asyncAction) => {
        setLoading(true);
        setError(null);
        try {
            await dispatch(asyncAction).unwrap();
        } catch (err) {
            setError(err.message);
        } finally {
            setLoading(false);
        }
    };

    return { execute, loading, error };
};

üíº Redux Interview Questions
Q1. Redux kya hai aur kyu use karte hain? +

Hinglish: Redux ek state management library hai jo application ka saara data ek jagah store karta hai. Jab bahut saare components ko same data chahiye hota hai, tab Redux use karte hain taaki prop drilling se bach sakein.

English Definition: Redux is a predictable state container for JavaScript applications. It helps manage the global state of an application by storing all state in a single centralized store, making state management more predictable and easier to debug.

Key Points:

  • Single source of truth
  • State is read-only
  • Changes made with pure functions
  • Predictable state updates
Q2. Redux ke main components kya hain? +

Hinglish: Redux ke 4 main parts hain - Store (jahan data rakha jata hai), Actions (kya karna hai ye batata hai), Reducers (data ko update karta hai), aur Dispatch (action ko bhejta hai).

English Definition:
  • Store: The centralized state container that holds the entire application state
  • Actions: Plain JavaScript objects that describe what happened
  • Reducers: Pure functions that specify how state changes in response to actions
  • Dispatch: Method to send actions to the store
Q3. Redux Toolkit kyu better hai old Redux se? +

Hinglish: RTK mein kam code likhna padta hai, setup easy hai, aur built-in features jaise Immer aur Thunk already included hain. Old Redux mein bahut zyada boilerplate code likhna padta tha.

English Definition: Redux Toolkit (RTK) is the official, opinionated, batteries-included toolset for efficient Redux development. It includes:
  • configureStore() - Simplified store setup
  • createSlice() - Reduces boilerplate
  • createAsyncThunk() - Easy async logic
  • Built-in Immer - Immutable updates made easy
  • Redux DevTools - Automatic integration
Q4. useSelector aur useDispatch hooks ka kya use hai? +

Hinglish: useSelector store se data read karne ke liye use hota hai, aur useDispatch actions bhejne ke liye use hota hai. Ye dono React-Redux ke hooks hain.

English Definition:
  • useSelector: A hook that allows you to extract data from the Redux store state using a selector function
  • useDispatch: A hook that returns a reference to the dispatch function from the Redux store

Q5. Reducer pure function kyu hona chahiye? +

Hinglish: Pure function isliye chahiye kyunki same input pe hamesha same output milna chahiye. Isse Redux predictable rehta hai aur debugging easy ho jati hai. Side effects nahi hone chahiye reducer mein.

English Definition: A pure function is a function that:
  • Always returns the same output for the same input
  • Has no side effects (no API calls, no mutations, no random values)
  • Doesn't modify its arguments
This makes Redux predictable, testable, and enables time-travel debugging. 
// ‚úÖ Pure Function (Good)
function counterReducer(state = 0, action) {
    if (action.type === 'INCREMENT') {
        return state + 1;  // Returns new value
    }
    return state;
}

// ‚ùå Impure Function (Bad)
function badReducer(state = 0, action) {
    if (action.type === 'INCREMENT') {

        fetch('/api/log');  // Side effect

    }

Q6. createSlice kya karta hai? +

Hinglish: createSlice ek function hai jo automatically actions aur reducer dono bana deta hai. Isme hum sirf reducers define karte hain aur actions automatically generate ho jate hain.

English Definition: createSlice is a function that accepts an initial state, an object of reducer functions, and a slice name. It automatically generates action creators and action types that correspond to the reducers and state.
Q7. Middleware kya hota hai Redux mein? +

Hinglish: Middleware ek layer hai jo action dispatch hone aur reducer tak pahunchne ke beech mein kaam karta hai. Iska use logging, async operations, ya error handling ke liye hota hai.

English Definition: Middleware provides a third-party extension point between dispatching an action and the moment it reaches the reducer. It's used for logging, crash reporting, performing asynchronous tasks, etc.
Q8. createAsyncThunk ka use kab karte hain? +

Hinglish: Jab API call karni ho ya koi async operation karna ho tab createAsyncThunk use karte hain. Ye automatically pending, fulfilled, aur rejected states handle kar leta hai.

English Definition: createAsyncThunk is used to handle asynchronous logic in Redux. It accepts an action type string and a callback function that returns a promise, and automatically dispatches pending/fulfilled/rejected actions based on the promise lifecycle.

// cartSlice.js - Complete implementation
import { createSlice } from '@reduxjs/toolkit';

const cartSlice = createSlice({
    name: 'cart',
    initialState: {
        items: [],
        totalQuantity: 0,
        totalAmount: 0,
        discount: 0
    },
    reducers: {
        addToCart: (state, action) => {
            const newItem = action.payload;
            const existingItem = state.items.find(item => item.id === newItem.id);
            
            if (existingItem) {
                existingItem.quantity++;
                existingItem.totalPrice += newItem.price;
            } else {
                state.items.push({
                    ...newItem,
                    quantity: 1,
                    totalPrice: newItem.price
                });
            }
            
            state.totalQuantity++;
            state.totalAmount += newItem.price;
        },
        removeFromCart: (state, action) => {
            const id = action.payload;
            const existingItem = state.items.find(item => item.id === id);
            
            if (existingItem) {
                state.totalQuantity -= existingItem.quantity;
                state.totalAmount -= existingItem.totalPrice;
                state.items = state.items.filter(item => item.id !== id);
            }
        },
        updateQuantity: (state, action) => {
            const { id, quantity } = action.payload;
            const item = state.items.find(item => item.id === id);
            
            if (item) {
                const diff = quantity - item.quantity;
                item.quantity = quantity;
                item.totalPrice = item.price * quantity;
                state.totalQuantity += diff;
                state.totalAmount += diff * item.price;
            }
        },
        applyDiscount: (state, action) => {
            state.discount = action.payload;
        },
        clearCart: (state) => {
            state.items = [];
            state.totalQuantity = 0;
            state.totalAmount = 0;
            state.discount = 0;
        }
    }
});

export const { 
    addToCart, 
    removeFromCart, 
    updateQuantity, 
    applyDiscount, 
    clearCart 
} = cartSlice.actions;

export const selectCartTotal = (state) => {
    const { totalAmount, discount } = state.cart;
    return totalAmount - (totalAmount * discount / 100);
};

export default cartSlice.reducer;
