Basic Types aur Type Annotations

TypeScript mein variables ko specific types assign kar sakte hain. Yeh compile time pe type checking enable karta hai.

JavaScript vs TypeScript Comparison:

üìÑ JavaScript (Dynamic)

// JavaScript - No type checking
let name = "John";
let age = 25;
let isStudent = true;

function greet(person) {
    return "Hello " + person.name;
}

// Runtime error possible
greet("string"); // TypeError!

üìò TypeScript (Static)

// TypeScript - Type annotations
let name: string = "John";
let age: number = 25;
let isStudent: boolean = true;

function greet(person: {name: string}): string {
    return "Hello " + person.name;
}

// Compile-time error prevention
// greet("string"); // Error caught!

Interfaces aur Type Aliases

Interfaces aur type aliases complex types define karne ke liye use hote hain. Yeh code ko more readable aur maintainable banate hain.

Interface vs Type Alias:

üî∑ Interface

interface User {
    id: number;
    name: string;
    email: string;
    isActive?: boolean; // Optional
}

// Interface extension
interface AdminUser extends User {
    permissions: string[];
}

// Implementation
const user: User = {
    id: 1,
    name: "John",
    email: "john@example.com"
};

üè∑Ô∏è Type Alias

type User = {
    id: number;
    name: string;
    email: string;
    isActive?: boolean; // Optional
};

// Union types
type Status = "pending" | "approved" | "rejected";

// Function types
type EventHandler = (event: Event) => void;

// Generic types
type ApiResponse<T> = {
    data: T;
    status: number;
};

Interview Questions

1. Basic: TypeScript kya hai aur JavaScript se kya different hai?

Answer: TypeScript JavaScript ka superset hai jo static typing add karta hai. Compile time pe JavaScript mein convert ho jaata hai.

2. Basic: Interface aur type alias mein kya difference hai?

Answer: Interface extend ho sakta hai aur declaration merging support karta hai. Type alias union types aur computed properties support karta hai.

3. Intermediate: Optional properties kaise define karte hain?

Answer: Property name ke baad ? lagakar: interface User { name: string; age?: number; }

4. Intermediate: Union types kya hain? Example do.

Answer: Multiple types allow karna: type Status = "loading" | "success" | "error";

5. Advanced: Generic types kya hain aur kab use karte hain?

Answer: Reusable components jo different types ke saath work kar saken: function identity<T>(arg: T): T { return arg; }
