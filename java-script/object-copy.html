üìö Basics - Object Reference vs Copy

Samjho yaar: JavaScript mein jab tum ek object banate ho, toh wo memory mein store hota hai. Jab tum us object ko kisi variable mein assign karte ho, toh actually wo object copy nahi hota, balki uska reference (address) copy hota hai. Matlab dono variables same object ko point karte hain!
Definition: In JavaScript, objects are reference types. When you assign an object to a variable, you're storing a reference (memory address) to that object, not the actual object itself.

let original = { name: "Rajesh", age: 25, city: "Mumbai" }; let copy = original; // Ye copy nahi hai, reference hai! copy.name = "Suresh"; // Original bhi change ho jayega! console.log(original.name); // Output: "Suresh" üò±

üìã Shallow Copy

Kya hai ye: Shallow copy matlab object ki first level properties copy ho jati hain, lekin agar koi nested object hai (object ke andar object), toh uska sirf reference copy hota hai. Matlab nested objects mein changes dono jagah reflect honge!
Definition: A shallow copy creates a new object and copies the values of the original object's properties. However, if a property value is a reference to another object, only the reference is copied, not the actual nested object.

üõ†Ô∏è Shallow Copy Kaise Banaye?

// Method 1: Spread Operator (...) let original = { name: "Amit", age: 30 }; let copy1 = { ...original }; // Method 2: Object.assign() let copy2 = Object.assign({}, original); // Method 3: Object.create() + Object.assign() let copy3 = Object.create( Object.getPrototypeOf(original), Object.getOwnPropertyDescriptors(original) );

üîç Deep Copy

Perfect solution: Deep copy matlab pura object copy ho jata hai, including all nested objects! Matlab original aur copy completely independent ho jate hain. Kisi mein bhi change karo, dusra affect nahi hoga! üéâ
Definition: A deep copy creates a new object and recursively copies all nested objects and their properties. The copied object is completely independent of the original, with no shared references.

üõ†Ô∏è Deep Copy Kaise Banaye?

// Method 1: JSON.parse + JSON.stringify (Simple but limited) let original = { name: "Priya", address: { city: "Pune" } }; let deepCopy1 = JSON.parse(JSON.stringify(original)); // Method 2: structuredClone() (Modern & Best) let deepCopy2 = structuredClone(original); // Method 3: Recursive Function (Custom) function deepClone(obj) { if (obj === null || typeof obj !== 'object') return obj; let clone = Array.isArray(obj) ? [] : {}; for (let key in obj) { if (obj.hasOwnProperty(key)) { clone[key] = deepClone(obj[key]); } } return clone; }

‚ö†Ô∏è JSON Method Ki Limitations

JSON.parse(JSON.stringify()) simple hai but ye kuch cheezein copy nahi kar sakta:
  • Functions - Functions lost ho jate hain
  • undefined - undefined values remove ho jati hain
  • Date objects - String ban jate hain
  • RegExp - Empty object ban jata hai
  • Infinity/NaN - null ban jate hain
  • Circular references - Error throw karta hai

üîß Object.assign()

Kya hai ye: Object.assign() ek built-in method hai jo ek ya zyada source objects ki properties ko target object mein copy kar deta hai. Ye bhi shallow copy karta hai, matlab nested objects ka reference copy hota hai!
Definition: Object.assign() copies all enumerable own properties from one or more source objects to a target object. It returns the modified target object. It performs a shallow copy.

üìù Syntax

Object.assign(target, source1, source2, ...sourceN) // Example: let target = { a: 1 }; let source = { b: 2, c: 3 }; let result = Object.assign(target, source); // target ab { a: 1, b: 2, c: 3 } hai

‚ö†Ô∏è Important Points

  • Shallow copy: Nested objects ka reference copy hota hai

  • Prototype chain: Inherited properties copy nahi hoti

üí° Kab Kya Use Karein?

‚úÖ Reference (=) Use Karo Jab:

  • Same object ko multiple variables mein chahiye
  • Memory save karni ho
  • Changes everywhere reflect hone chahiye
let config = { theme: "dark" }; let appConfig = config; // Dono same object point karenge

‚úÖ Shallow Copy Use Karo Jab:

  • Simple flat objects copy karni ho
  • Nested objects nahi hain
  • Fast performance chahiye
  • Objects merge karne hain
let user = { name: "Raj", age: 25 }; let copy = { ...user };

‚úÖ Deep Copy Use Karo Jab:

  • Complex nested objects ho
  • Complete independence chahiye
  • Arrays of objects ho
  • State management mein (Redux, etc.)
let data = { user: { address: { city: "Delhi" } } }; let copy = structuredClone(data);

‚úÖ Object.assign() Use Karo Jab:

  • Multiple objects merge karni ho
  • Default values set karni ho
  • Options/config objects handle karni ho
  • Older browser support chahiye
let defaults = { a: 1 }; let options = { b: 2 }; let final = Object.assign({}, defaults, options);

function deepClone(obj, hash = new WeakMap()) { // Null ya primitive values if (obj === null || typeof obj !== 'object') { return obj; } // Circular reference check if (hash.has(obj)) { return hash.get(obj); } // Date object if (obj instanceof Date) { return new Date(obj.getTime()); } // RegExp object if (obj instanceof RegExp) { return new RegExp(obj); } // Array if (Array.isArray(obj)) { const arrCopy = []; hash.set(obj, arrCopy); obj.forEach((item, index) => { arrCopy[index] = deepClone(item, hash); }); return arrCopy; } // Object const objCopy = {}; hash.set(obj, objCopy); Object.keys(obj).forEach(key => { objCopy[key] = deepClone(obj[key], hash); }); return objCopy; }
