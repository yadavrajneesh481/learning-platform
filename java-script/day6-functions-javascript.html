üìöWhat are Functions?

Functions are reusable blocks of code that perform a specific task. They help organize code, avoid repetition, and make programs more maintainable.

Functions code ke reusable blocks hain jo ek specific task perform karte hain. Ye code ko organize karne, repetition avoid karne, aur programs ko maintainable banane mein help karte hain.

Analogy: Think of a function like a recipe! You write the recipe once (define function), and then you can cook the dish anytime by following that recipe (calling function). You don't need to remember all steps every time!
Key Definition: A function is a named block of code that can be executed whenever needed by calling its name.

Why Use Functions?

  • Reusability: Write once, use multiple times
  • Organization: Break complex problems into smaller parts
  • Maintainability: Easy to update and debug
  • Abstraction: Hide complex logic behind simple names

üîÄFunction Declaration vs Expression

JavaScript has multiple ways to create functions. The two main types are Function Declarations and Function Expressions.

JavaScript mein functions banane ke kai tarike hain. Do main types hain: Function Declarations aur Function Expressions.

1. Function Declaration

// Function Declaration

    return a + b;
}

// Can be called before declaration (Hoisting)

function multiply(a, b) {
    return a * b;
}

2. Function Expression

// Function Expression (Anonymous)
const subtract = function(a, b) {
    return a - b;
};

console.log("10 - 3 = " + subtract(10, 3));

// Named Function Expression
const divide = function divideNumbers(a, b) {
    if (b === 0) {
        return "Cannot divide by zero!";
    }
    return a / b;
};

console.log("20 / 4 = " + divide(20, 4));
console.log("10 / 0 = " + divide(10, 0));

‚û°Ô∏èArrow Functions

Arrow functions are a shorter syntax for writing function expressions. Introduced in ES6, they provide a more concise way to write functions.

Arrow functions function expressions likhne ka ek shorter syntax hai. ES6 mein introduce hua, ye functions likhne ka zyada concise tarika provide karta hai.

Analogy: Arrow functions are like shortcuts on your phone - they do the same thing as the full app, but faster and with less typing!

Basic Syntax

// Traditional Function Expression
const func1 = function(param) {
    return param * 2;
};

// Arrow Function
const func2 = (param) => {
    return param * 2;
};

// Arrow Function with Implicit Return (one-liner)
const func3 = param => param * 2;

Example: Different Arrow Function Styles

// No parameters
const sayHello = () => "Hello, World!";
console.log(sayHello());

// One parameter (parentheses optional)
const square = num => num * num;
console.log("Square of 5: " + square(5));

// Multiple parameters
const add = (a, b) => a + b;
console.log("10 + 20 = " + add(10, 20));

// Multiple lines (need curly braces and return)
const greetUser = (name, age) => {
    const message = `Hello ${name}, you are ${age} years old!`;
    return message;
};
console.log(greetUser("Rahul", 25));

Implicit Return

Implicit return means the function automatically returns the expression without using the return keyword. Works only with single-line arrow functions.

Implicit return matlab function automatically expression ko return kar deta hai bina return keyword use kiye. Sirf single-line arrow functions ke saath kaam karta hai.

// Explicit return (with return keyword)
const multiply1 = (a, b) => {
    return a * b;
};

// Implicit return (without return keyword)
const multiply2 = (a, b) => a * b;

console.log("Explicit: " + multiply1(5, 3));
console.log("Implicit: " + multiply2(5, 3));

// Implicit return with object (need parentheses)
const createPerson = (name, age) => ({
    name: name,
    age: age
});

console.log(createPerson("Priya", 22));
Tip: To implicitly return an object, wrap it in parentheses: () => ({ key: value })

üì¶Rest Parameters

Rest parameters allow a function to accept an indefinite number of arguments as an array. Use three dots (...) before the parameter name.

Rest parameters se function unlimited arguments ko array ke roop mein accept kar sakta hai. Parameter name se pehle teen dots (...) use karo.

Analogy: Like a shopping bag that can hold any number of items - you don't need to know in advance how many items you'll buy!

Example: Rest Parameters

// Function with rest parameters
function sum(...numbers) {
    console.log("Received " + numbers.length + " numbers");
    let total = 0;
    for (let num of numbers) {
        total += num;
    }
    return total;
}

console.log("Sum of 1, 2, 3: " + sum(1, 2, 3));
console.log("Sum of 10, 20, 30, 40: " + sum(10, 20, 30, 40));
console.log("Sum of 5, 15, 25, 35, 45, 55: " + sum(5, 15, 25, 35, 45, 55));

// Combining regular and rest parameters
function introduce(name, age, ...hobbies) {
    console.log(`Name: ${name}, Age: ${age}`);
    console.log("Hobbies: " + hobbies.join(", "));
}

introduce("Rahul", 25, "Cricket", "Reading", "Coding", "Gaming");

Example: Finding Maximum with Rest Parameters

function findMax(...numbers) {
    if (numbers.length === 0) {
        return "No numbers provided!";
    }
    
    let max = numbers[0];
    for (let num of numbers) {
        if (num > max) {
            max = num;
        }
    }
    return max;
}

console.log("Max of 5, 12, 8, 3: " + findMax(5, 12, 8, 3));
console.log("Max of 45, 23, 89, 12, 67: " + findMax(45, 23, 89, 12, 67));

// Using with arrow function

Important: Rest parameter must be the last parameter in the function definition!

Function Declaration 
// Generate factorial
function factorial(n) {
    if (n < 0) return "Factorial not defined for negative numbers";
    if (n === 0 || n === 1) return 1;
    
    let result = 1;
    for (let i = 2; i <= n; i++) {
        result *= i;
    }
    return result;
}

// Generate Fibonacci sequence
function fibonacci(n) {
    if (n <= 0) return [];
    if (n === 1) return [0];
    
    const fib = [0, 1];
    for (let i = 2; i < n; i++) {
        fib.push(fib[i - 1] + fib[i - 2]);
    }
    return fib;
}

// Generate prime numbers up to n
function getPrimes(limit) {
    const primes = [];
    for (let num = 2; num <= limit; num++) {
        let isPrime = true;
        for (let i = 2; i <= Math.sqrt(num); i++) {
            if (num % i === 0) {
                isPrime = false;
                break;
            }
        }
        if (isPrime) primes.push(num);
    }
    return primes;
}

// Test sequence generators

console.log("First 10 Fibonacci numbers: " + fibonacci(10));
console.log("Prime numbers up to 30: " + getPrimes(30));

function isPrime1(num) {
    if (num <= 1) return false;
    if (num <= 3) return true;
    
    for (let i = 2; i < num; i++) {
        if (num % i === 0) return false;
    }
    return true;

// Method 2: Optimized (check up to square root)
function isPrime2(num) {
    if (num <= 1) return false;
    if (num <= 3) return true;
    if (num % 2 === 0 || num % 3 === 0) return false;
    
    for (let i = 5; i * i <= num; i += 6) {
        if (num % i === 0 || num % (i + 2) === 0) {
            return false;
        }
    }
    return true;
}

const reverseString1 = str => str.split('').reverse().join('');

// Method 2: Using loop
function reverseString2(str) {
    let reversed = '';
    for (let i = str.length - 1; i >= 0; i--) {
        reversed += str[i];
    }
    return reversed;
}

// Method 3: Using reduce
const reverseString3 = str => 
    str.split('').reduce((rev, char) => char + rev, '');

// Method 4: Recursive
function reverseString4(str) {
    if (str === '') return '';
    return reverseString4(str.substr(1)) + str[0];
}

Function: A reusable block of code that performs a specific task and can be called by name.
Parameter: A variable listed in the function definition that acts as a placeholder for values.
Argument: The actual value passed to a function when it is called.
Return Value: The value that a function sends back to the code that called it using the return statement.
Arrow Function: A concise syntax for writing function expressions using the => operator.
Default Parameter: A parameter that has a default value if no argument is provided.
Rest Parameter: A parameter that collects all remaining arguments into an array using ...paramName syntax.
Implicit Return: Automatic return of an expression in arrow functions without using the return keyword.
